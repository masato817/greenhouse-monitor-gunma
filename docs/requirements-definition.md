## 1. はじめに

### 1.1 文書の目的

本文書は、施設園芸環境モニタリングシステムの開発に必要な要件を定義する。
開発者、運用担当者、および関係者が共通の理解を持つための基準文書とする。

### 1.2 対象読者

- システム開発者
- 運用管理者
- 栽培管理責任者
- 経営者

### 1.3 プロジェクト背景

サングレイス（Moss Farm Sangrace）は、静岡県菊川市で4棟・16,000㎡の大規模トマト栽培を行っている。現在、環境データの確認は手動でプロファインダー・プロファームにログインして行っており、以下の課題がある：

| 課題 | 影響 |
|------|------|
| データ確認に時間がかかる | 1回あたり5-10分の作業時間 |
| 異常値の発見が遅れる | 温度上昇等による作物へのダメージ |
| 夜間・休日の監視が困難 | 異常発生時の対応遅延 |
| データが蓄積されない | 過去との比較・分析が困難 |

### 1.4 プロジェクト目標

1. 環境データの自動取得・蓄積
2. Webダッシュボードによる一覧表示
3. 異常値の早期検知とLINE通知
4. 栽培管理の効率化
5. データに基づく栽培改善の基盤構築

## 3. 用語定義

| 用語 | 定義 |
|------|------|
| プロファインダー | デルフィー社製の環境制御システム。温度・湿度等を監視 |
| プロファーム | 誠和社製の統合環境制御システム |
| 環境データ | 温度、湿度、CO2濃度、日射量、飽差等の栽培環境測定値 |
| 閾値 | 異常判定の基準となる上限値・下限値 |
| 飽差（VPD） | 空気が含むことのできる水蒸気量の余裕度。植物の蒸散に影響 |
| EC値 | 養液の電気伝導度。肥料濃度の指標 |
| スプシ | Googleスプレッドシートの略称 |
| スクレイピング | Webページからデータを自動抽出する技術 |
| Puppeteer | Googleが開発したブラウザ自動操作ライブラリ |
| 静的サイト | サーバー処理不要で表示できるHTML/CSS/JSのみのWebサイト |
| GitHub Pages | GitHubが提供する無料の静的サイトホスティングサービス |

## 4. 機能要件

### 4.1 F-01: データ取得機能

#### 4.1.1 概要
プロファインダーおよびプロファームのWebシステムにログインし、環境データを自動取得する。

#### 4.1.2 詳細要件

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-01-01 | プロファインダーへの自動ログインができること | 必須 |
| F-01-02 | プロファームへの自動ログインができること | 必須 |
| F-01-03 | ログイン認証情報は環境変数で管理すること | 必須 |
| F-01-04 | ページ読み込み失敗時は自動リトライすること（最大3回） | 必須 |
| F-01-05 | デバッグ用にスクリーンショット保存ができること | 任意 |
| F-01-06 | ヘッドレスモード（画面非表示）で動作すること | 必須 |

#### 4.1.3 取得データ項目

**共通項目（全ハウス）**

| No. | 項目名 | 単位 | 必須 | 備考 |
|-----|--------|------|------|------|
| 1 | 日時 | - | ○ | データ取得時刻 |
| 2 | 気温 | ℃ | ○ | 現在の気温 |
| 3 | CO2濃度 | ppm | ○ | 光合成に影響 |
| 4 | 相対湿度 | % | ○ | 相対湿度 |
| 5 | 日射量 | W/m² | ○ | 現在の日射量 |
| 6 | **積算日射量** | **MJ/m²** | **○** | **本日の積算日射量（灌水目安計算に使用）** |
| 7 | 飽差 | g/m³ | ○ | VPD（飽差） |
| 8 | 本日最高気温 | ℃ | ○ | 当日の最高気温 |
| 9 | 本日最低気温 | ℃ | ○ | 当日の最低気温 |
| 10 | 24H移動平均気温 | ℃ | ○ | 過去24時間の平均気温 |

**プロファインダー追加項目（2号棟・3号棟・4号棟）**

| No. | 項目名 | 単位 | 必須 | 備考 |
|-----|--------|------|------|------|
| 10 | 前日昼平均気温(A) | ℃ | △ | 時間帯別平均 |
| 11 | 前日夜平均気温(B) | ℃ | △ | 時間帯別平均 |
| 12 | 当日昼平均気温(C) | ℃ | △ | 時間帯別平均 |
| 13 | 当日夜平均気温(D) | ℃ | △ | 時間帯別平均 |
| 14 | 昼夜温較差(A-B) | ℃ | △ | 昼夜の温度差 |
| 15 | 48H移動平均気温 | ℃ | △ | 過去48時間の平均 |
| 16 | 72H移動平均気温 | ℃ | △ | 過去72時間の平均 |

※ ○：必須、△：取得可能な場合

#### 4.1.4 データ取得元の詳細

**プロファインダー（2号棟・3号棟・4号棟）**

| 項目 | 内容 |
|------|------|
| URL | プロファインダークラウド |
| 対象ハウス | 2号棟、3号棟、4号棟 |
| 選択方法 | ドロップダウンリストで切り替え |
| 認証 | ユーザー名・パスワード |

```
[操作フロー]
1. ログイン
2. 環境測定データ画面を開く
3. 装置ドロップダウンで「サングレイス2号棟」を選択
4. データ取得
5. ドロップダウンで「サングレイス3号棟」に切り替え
6. データ取得
7. ドロップダウンで「サングレイス4号棟」に切り替え
8. データ取得
```

**プロファーム（1号棟）**

| 項目 | 内容 |
|------|------|
| URL | プロファーム管理画面 |
| 対象ハウス | 1号棟 |
| センサー | **センサ1を使用** |
| 認証 | ユーザー名・パスワード |

```
[操作フロー]
1. ログイン
2. リアルタイムモニタ画面を開く
3. ハウス内状況からセンサ1のデータを取得
4. 外気象から日射量を取得
5. 明期/暗期平均気温を取得
```

### 4.4 F-04: 異常検知機能

#### 4.4.1 概要
取得した環境データを閾値と比較し、異常値を検出する。

#### 4.4.2 詳細要件

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-04-01 | 現在時刻に該当する閾値を自動選択すること | 必須 |
| F-04-02 | 上限超過・下限未満を検出すること | 必須 |
| F-04-03 | 複数項目の同時異常を検出できること | 必須 |
| F-04-04 | 異常レベル（warning/critical）を判定すること | 任意 |
| F-04-05 | 異常履歴をスプシに記録すること | 必須 |

#### 4.4.3 異常判定ロジック

```
IF 実測値 < 下限値 THEN
    異常（下限超過）
ELSE IF 実測値 > 上限値 THEN
    異常（上限超過）
ELSE
    正常
END IF
```

### 4.5 F-05: LINE通知機能（異常時のみ）

#### 4.5.1 概要
LINE Messaging APIを使用して、**異常検知時のみ**アラートを送信する。
定期レポートはWebダッシュボードで確認するため、LINEは緊急通知専用とする。

#### 4.5.2 詳細要件

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-05-01 | LINE Messaging APIを使用すること | 必須 |
| F-05-02 | **異常検知時のみ**アラートを送信すること | 必須 |
| F-05-03 | 異常項目・場所・数値を含むメッセージを送信すること | 必須 |
| F-05-04 | ダッシュボードへのリンクを含めること | 必須 |
| F-05-05 | 同一異常の連続通知を抑制すること（クールダウン機能） | 必須 |
| F-05-06 | グループへの送信に対応すること | 任意 |

#### 4.5.3 通知メッセージ例

```
🚨 環境異常アラート

【3号棟】温度が基準値を超過
　現在値: 36.2℃
　基準値: 18〜32℃

📊 詳細はダッシュボードで確認
https://sangrace.github.io/greenhouse-monitor/
```

#### 4.5.4 通知抑制ルール

| ルール | 内容 |
|--------|------|
| クールダウン | 同一ハウス・同一項目の異常は30分間再通知しない |
| 復旧通知 | 異常が解消された場合は復旧通知を送信（任意） |

### 4.6 F-06: 定期実行機能

#### 4.6.1 概要
設定された間隔で自動的にデータ取得・ダッシュボード更新を実行する。

#### 4.6.2 詳細要件

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-06-01 | 実行間隔を環境変数で設定できること | 必須 |
| F-06-02 | **日中（6:00-18:00）は15分間隔**で実行すること | 必須 |
| F-06-03 | **夜間（18:00-6:00）は1時間間隔**で実行すること | 必須 |
| F-06-04 | 起動時に1回即時実行すること | 必須 |
| F-06-05 | エラー発生時もプロセスを継続すること | 必須 |
| F-06-06 | 実行ログを出力すること | 必須 |

#### 4.6.3 実行スケジュール

| 時間帯 | 間隔 | 1日の実行回数 | 備考 |
|--------|------|--------------|------|
| 06:00-18:00 | 15分 | 48回 | 日中（作業時間帯） |
| 18:00-06:00 | 60分 | 12回 | 夜間（監視のみ） |
| **合計** | - | **60回/日** | |

### 4.7 F-07: Webダッシュボード機能 ★メイン機能

#### 4.7.1 概要
環境データを静的HTMLとして生成し、GitHub Pagesで無料公開する。
スマホ・PC・タブレットすべてで閲覧可能なレスポンシブデザインとする。

#### 4.7.2 なぜ静的サイトか

| 方式 | 費用 | 更新方法 | 採用 |
|------|------|----------|------|
| リアルタイム（WebSocket） | 月数千円〜 | サーバー常時稼働 | ✗ |
| 動的サイト（Node.js等） | 月数百円〜 | サーバー必要 | ✗ |
| **静的サイト（HTML生成）** | **無料** | 定期的にHTML更新 | **◎** |

#### 4.7.3 詳細要件

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-07-01 | 4ハウス分のデータを1画面に表示すること | 必須 |
| F-07-02 | 各ハウスの温度・湿度・CO2等を同時表示すること | 必須 |
| F-07-03 | 正常値は緑、警告は黄、異常は赤で色分けすること | 必須 |
| F-07-04 | 最終更新日時を表示すること | 必須 |
| F-07-05 | スマホ・PC・タブレットで見やすいレスポンシブデザインとすること | 必須 |
| F-07-06 | GitHub Pagesで無料ホスティングすること | 必須 |
| F-07-07 | 過去24時間のグラフを表示すること | 任意 |
| F-07-08 | ブックマーク/ホーム画面追加に対応すること（PWA対応） | 任意 |

## 10. 制約条件

### 10.1 技術的制約

| 制約 | 理由 |
|------|------|
| スクレイピング方式 | プロファインダー/プロファームにAPIがないため |
| Node.js必須 | Puppeteerの動作要件 |
| インターネット接続必須 | 外部サービスへのアクセスに必要 |
| 静的サイトのみ | 無料ホスティングの制約 |
| リアルタイム更新不可 | 静的サイトのため（15分間隔で更新） |

### 10.2 運用上の制約

| 制約 | 内容 |
|------|------|
| 対象サイトの変更 | UI変更時はセレクタの修正が必要 |
| LINE通知数 | 無料枠は月200通まで（異常時のみなら十分） |
| 認証情報管理 | セキュリティ上、厳重に管理が必要 |
| ダッシュボード公開範囲 | URLを知っていれば誰でもアクセス可能 |

### 10.3 法的制約

| 制約 | 対応 |
|------|------|
| スクレイピングの利用規約 | 自社データの取得目的であり、過度な負荷をかけない |
| 個人情報保護 | 環境データのみ取得、個人情報は扱わない |
